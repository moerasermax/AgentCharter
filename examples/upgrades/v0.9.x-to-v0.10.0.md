## 5. 已有 `.git/hooks/pre-commit` 衝突？

若你 / 你的專案已有自訂 pre-commit hook（如 `husky` / lint hook），install 時會：

1. 把舊 hook 備份到 `.git/hooks/pre-commit.charter-backup-<timestamp>`
2. 寫入 charter shim
3. 提示你手動合併備份

合併方式（範例）：

```bash
# 編輯 .git/hooks/pre-commit、把舊邏輯加在 exec 之前：
#!/bin/sh
# 舊 hook 邏輯（如 lint）
bash .git/hooks/pre-commit.charter-backup-XXXXX || exit 1

# charter checks
exec bash agent-commons/_config/hooks/charter-commit-checks.sh "$@"
```

## 常見問題

**Q: 我多人團隊、每個人都要自己跑 install-git-hooks.sh 嗎？**
A: 是。`.git/hooks/` 是 local-only、git 設計如此。但因為實際邏輯在 `agent-commons/_config/hooks/`（入 git），所有人 pull 後跑一次 install 就同步、之後 charter 升版只需 `--update`。

**Q: 跨多版升（例如 v0.7.x 直接升 v0.10.0）怎麼辦？**
A: 找最接近 v0.10.0 的 walkthrough 當入口（例如 v0.7.x → v0.8.0 → v0.9.0 → v0.10.0），逐版讀重點即可。架構升維是累積的、跳一版不會壞。

**Q: 想暫時繞過 hook（例如緊急修補）？**
A: `git commit --no-verify`（git 既有逃生口）。但 AI 自主繞過 = F1 結構性繞路（依 commit-hook-spec §6）。
