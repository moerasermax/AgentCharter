## 0. 為什麼這個 walkthrough 重要

v0.9.0 是 charter 的**紀律完整性 + AI 自我覺察升維**關鍵 MINOR — 不只新加 4 條 condition + 新範本 + 新 preset，更是**完成 v0.7.3 北極星「不讓 user 記」對採用方 + AI 雙邊閉環**的最後一塊拼圖。

對既有採用方而言：

| 採用方狀態 | v0.9.0 影響 |
|---|---|
| **A. 全合規 + 已有 reflection 文化**（自發在 `roles/<role>/reflections/` 累積反省）| doctor §3.11 W1101/W1102 不會觸發；純擴增升版 |
| **B. 全合規但未啟動個體學習迴圈**（`roles/<role>/reflections/` 為空 / 不存在）| doctor §3.11 抓 **W1101**（警告）；升版前須跑 reflection 補建 prompt |
| **C. failure_mode_log 有 F-mode 命中但無對應 reflection 個體層 entry**（雙寫紀律首次落地）| doctor §3.11 抓 **W1102**（警告）；升版前須跑補寫 reflection prompt |
| **D. 想試 essential preset（v0.9.0 新加、3-5 條 core、< 5k init token）** | 純切 preset、無 migration |

→ 本 walkthrough 對 A/B/C/D 四類採用方分別給 step-by-step 流程 + 每步可貼的 AI prompt。

### 0.1 v0.9.0 對採用方的核心紀律升維

| 升維軸 | v0.8.2（before）| v0.9.0（after）|
|---|---|---|
| **AI 違規後處理** | 寫集體層 `state/failure_mode_log.md` | **雙寫**：集體層 + 個體層 `roles/<role>/reflections/<date>_<f-mode>_<short>.md` |

| **棄用場景** | 無條款 | `core/adoption-lifecycle.md` + `tools/uninstall-spec.md`「**保留最後的溫柔**」精神 |
| **condition mutability** | frontmatter scaffold（v0.7.1）| **紀律本體條款化**（三層 mutability + 3-strike 刪除協議 + user-initiated consolidation）|
| **採用門檻** | minimal 9/19 | **essential 3-5/19 < 5k init token**（探索期專案首選）|

## 3. 升版 5 步流程（每步給 AI 的 prompt 模板）

> ⚠️ **signal #21 紀律對齊**：每步都是「**user 動作 = 貼一個 prompt**」、AI 動作 = 完成檔修。**不要求 user 自己編輯 yaml/markdown**（除非 prompt 內明示）。

### Step 1：拉 charter 最新版

**user 動作**：開終端機、跑：

```powershell
cd $env:USERPROFILE/.agentcharter   # PowerShell；macOS/Linux 用 cd ~/.agentcharter
git pull origin main
git log --oneline -3
```

**驗證**：git log 含 v0.9.0 對應 commit + tag `v0.9.0`。

> 💡 **跨版本場景**（user LIVE 實證紀律提示、dogfood signal #29 LIVE 實證）：若你停在 v0.7.5 / v0.8.0 / v0.8.1、`git pull` 後 git log 會直接顯示 v0.9.0 + 之後 PATCH — charter 跨版本自動向下兼容、直接走本 walkthrough、profile.yaml `charter_version` 改到 `"0.9.0"`（甚至更新 PATCH）即可。詳見 §5。

### Step 2：跑 doctor pre-check（抓 W1101/W1102）

**user 動作**：切回專案 working dir、貼以下 prompt 給你的 AI（PM Gemini / Engineer Claude 任一皆可）：

```
我要從 charter v0.8.2 升 v0.9.0。請依 ~/.agentcharter/tools/doctor-spec.md

  對每個 _role.md 內 status 為 ACTIVE 的角色：
  (1) roles/<role>/reflections/ 目錄存在 + 至少一個 reflection 檔 → 缺 W1101
  (2) state/failure_mode_log.md 內 F-mode 命中 entry 是否有對應
      reflections/<date>_<f-mode>_*.md 個體層 entry → 不對應 W1102
  (3) reflection 檔 frontmatter 完整（date / role / vendor / status /
      violations 五欄）→ 缺欄位 E1103

完成後請貼出：
(a) doctor 完整 stdout
(b) 若有 W1101/W1102/E1103、列出每項對應的修補建議（不要直接動手修、只列建議）

**預期結果分支**：

| 結果 | 你是哪類採用方 | 下一步 |
|---|---|---|
| **0 ERROR / 0 WARN** | A 類（全合規 + 已有 reflection 文化）| 跳到 [Step 4](#step-4升-charter_version) |
| **W1101**（reflections/ 缺）| B 類（未啟動個體學習迴圈）| 進 [Step 3.B](#step-3b啟動個體學習迴圈b-類採用方) |
| **W1102**（雙寫漏對應）| C 類（failure_mode_log 有 F-mode 命中但無對應個體層）| 進 [Step 3.C](#step-3c補寫個體層-reflection-雙寫c-類採用方) |
| **E1103**（frontmatter 不全）| 既有 reflection 檔不合規 | 進 [Step 3.E](#step-3e修補-reflection-frontmattere1103) |
| 其他 ERROR | 罕見、停下回報 | — |

```

   學習迴圈起點」說明）

### Step 5：自具象化（八步驟） + 跑 `/charter-upgrade-verify`
